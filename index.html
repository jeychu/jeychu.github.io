Linux Kernel

Posted on 2019-02-20

如何学习内核：

  • 以应用为目的来学习，边学边用，学以致用
  • 以培养内核的阅读能力为主要目标
  • 阅读和修改真实的代码，而非只是阅读书籍
  • 需要不断的更新和学习，因为内核的策略和方法一直在不断的变化着

内核基础层

内核中提供基础服务的部分

  • 代码总量很少，但难度高
  • 提供基本稳定的API
  • 数据结构
  • 内核同步机制
  • 内存管理和任务调度

  • 学习方法：

    • 以API接口学习为主要内容，不细究实现
    • 日后应该加强基础层的研读

      内核应用层

      建立在基础层之上，提供应用服务的部分

  • 占代码的90%以上，数量多，重复性强

  • 接口经常变化
  • 文件系统
  • 设备和驱动
  • 网络
  • 学习方法：
    • 文件系统是应用层的基础和灵魂
    • 从文件系统入手，驻步扩展
    • 以架构为核心，举一反三
    • 设备和驱动不依赖具体设备，以通用架构为主

最终目的：

  • 独立思考，独立学习，成为在该领域有影响力的专家
  • 在代码中寻找答案
  • 这个课程就是要形成一个起点，从这个起点出发，不断的学习、思考、实践、分享

内核编译：

  • 编译的实质：
    • 基于头文件和c文件产生对象文件
    • 将所有的对象文件链接起来，生成可执行文件
  • 内核编译的核心问题：make文件和配置文件

Nginx modules

Posted on 2019-02-17

HTTP 处理请求的11个阶段

阶段 模块
post_read realip
server_rewrite rewrite
find_config
rewrite rewrite
post_rewrite
pre access limit_conn,limit_req
access auth_basic,access,auth_request
post_access
pre content try_files
content index,autoindex,concat
log access_log

使用模块的前提：模块需要先编译进Nginx的二进制文件。
对于每个模块，我们需要了解：

  • 该模块提供哪些配置项
  • 模块何时被使用
  • 该模块提供了哪些变量
