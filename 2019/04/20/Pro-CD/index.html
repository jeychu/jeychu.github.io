Elements of CD

What is CD?

CD practices can include more or less the following entities:

  • a good branching strategy
  • a working ci process
  • distributed builds
  • automated testing
  • distributed or parallel testing
  • automated and quick environment provisioning
  • automated code promotion

CI

A robust CI can be defined by:

A codable CI pipeline

A CI pipeline is a set of sequential or parallel jobs.

To maintain the pipelines effectively, Jenkins and Gitlab come up with the concept of pipeline as a code.

In Jenkins, the file that stores the pipeline script is referred to as Jenkinsfile.

Jenkinsfile’s abilities:

  • Jenkinsfile can be a version control along with your source code
  • jenkinsfile is easily shareable
  • developers can themselves define what a jenkinsfile should do
  • you can have different jenkinsfile for different branches

How to Use Jenkinfile?

Jenkinsfile is stored along with the source code inside a version control system.

Whenever there is a code commit on the version control system, the following steps take place:

  • A source control webhook is sent to Jenkins.
  • The Jenkins pipeline Job is triggered on receiving the webhook condition.
  • The Jenkins pipeline Job downloads the latest source code as well as the Jenkinsfile from the version control system.
  • Jenkins reads the Jenkinsfile and executes the pipeline steps accordingly.

How to use a Dockerfile with Jenkins?

One dockerfile per environment:

  • Dockerfile for build & Unit test
  • Dockerfile for build & Integration test
  • Dockerfile for Acceptance testing
  • Dockerfile for Performance testing
  • Dockerfile for end-to-end testing

All these files can be kept under a version control system along with your source code.

Whenever there is a code commit on the version control system, the following steps take place:

  • A version control webhook is sent to Jenkins
  • The Jenkins pipeline Job gets triggered on receicing the webhook
  • The Jenkins pipeline Job downloads the latest source code as well as the Jenkinsfile and the set of Dockerfies from the version control system
  • Jenkins reads the Jenkinsfile and executes the pipeline steps accordingly
  • The first step inside the Jenkinsfile is to build all the required Docker images using the Dockerfiles
  • With the required Docker images built, Jenkins can now perform various pipeline steps on the respective Docker containers that are spawned using the Docker images.

How to use kubernetes with Jenkins?

More than one Docker host.
